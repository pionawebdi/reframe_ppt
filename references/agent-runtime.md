# Agent 런타임 브리프 — Pentagram처럼 브랜드 적용하기

## 핵심 원칙

이 스킬은 브랜드 시스템을 *적용*하는 것이지, *창작*하는 것이 아닙니다. 브랜드는 반드시 사용자가 제공해야 합니다. 브랜드 입력값이 불완전한 경우, agent는 임의로 추정하지 않고 사용자에게 요청합니다.

## 생성 전에 agent가 확보해야 할 컨텍스트

> 덱을 생성하기 전에, agent는 아래의 각 항목을 파악해야 합니다.
> - 사용자의 이전 메시지에 이미 해당 항목이 포함되어 있다면 그것을 사용하고, 다시 묻지 마세요.
> - 사용자가 제시한 주제로부터 합리적으로 추론 가능한 항목은 추론하고, 슬라이드 2에 인라인으로 가정 사항을 명시하세요.
> - 누락되어 있고 추론도 불가능한 항목만 질문하되 — 한 번에 하나씩 핵심 질문만 하고, 질문 목록을 나열하지 마세요.

1. **브랜드 시스템 입력값** — 팔레트(주색 + 보조색 + 기능색), 타이포그래피(디스플레이 + 본문 + 굵기), 로고(잠금 형식 + 여백 규칙), 그리드(컬럼 수 + 거터), 사진 처리 방식(색상 보정 + 크롭 비율), 차트 팔레트, 전환 언어. 누락된 입력값은 표시되며, agent는 임의로 만들지 않습니다.
2. **기존 덱** — 슬라이드 수, 형식, 그리고 외부 목적을 한 줄로 요약한 내용.
3. **대상 청중** — Fortune 500 RFP / 투자사 파트너 / 업계 컨퍼런스 / 전사 타운홀. 청중에 따라 브랜드 적용 방식의 결이 다릅니다.
4. **브랜드 제공 사진** — 사용자에게 브랜드 이미지 라이브러리가 있습니까? 없다면, agent는 브랜드의 색상 보정 기준에 맞게 재구성된 플레이스홀더 스튜디오 사진을 사용합니다.
5. **차트 팔레트 사양** — 브랜드에서 데이터 시각화 사양을 공개하고 있습니까? 있다면 모든 차트에 상속하고, 없다면 브랜드 주색을 사용한 단색 막대 차트를 기본으로 사용합니다.
6. **필수 법적/컴플라이언스 요소** — 면책 조항, 규제 마크, 파트너 공동 브랜드 잠금 형식. agent는 이 요소들을 파워포인트 구석 스티커 방식이 아닌, 브랜드에 맞는 올바른 위치에 적용합니다.

## 생성 중 필수 체크

- 7개의 브랜드 시스템(로고, 타이포그래피, 팔레트, 그리드, 사진, 차트, 전환) 모두가 모든 슬라이드에 상속됩니다. Brand-Decision Map은 슬라이드별, 시스템별로 상속된 결정 사항을 기록합니다.
- 전체 덱에서 사용하는 서체 굵기는 최대 세 가지로 제한합니다 (Vignelli / Pentagram의 절제 원칙).
- 타이포그래피 조합은 덱 전체에서 일관되게 유지됩니다 — 브랜드가 지정한 디스플레이 + 본문 조합 외의 폰트는 어떤 슬라이드에도 사용하지 않습니다.
- 모든 차트는 브랜드 팔레트를 상속합니다. Excel 기본 차트 색상(Office 파랑/주황)은 재작성됩니다.
- 기존 이미지는 브랜드의 색상 보정 기준에 맞게 재구성됩니다 (예: 브랜드가 지정한 경우 주색 10% 오버레이 적용). 스톡 사진 특유의 이미지("악수", "노트북 앞 다양한 팀원")는 교체하거나 제거합니다.
- 로고는 브랜드 사양에 맞는 크기와 여백으로 배치됩니다. 표지에 과도하게 크게 배치된 로고는 축소됩니다.
- Brand-Decision Map 부록은 완성된 상태로 포함되어야 합니다 — 슬라이드별 × 7개 시스템별로 한 행씩 구성됩니다.
- 모든 외부 주장에는 원본 덱의 출처 인용이 유지됩니다.
- 팔레트는 사용자가 브랜드 팔레트를 제공하지 않은 경우에 한해 고대비 쿨톤(네이비 `#0F2540` + 일렉트릭 블루 `#0066FF` + 화이트 `#FFFFFF`)을 기본값으로 사용합니다. 크림/아이보리/양피지/웜 베이지(`#F5EFE3`, `#F7F3EB`, `#FAF7F2`)는 기본값으로 사용하지 않으며 — 사용자의 브랜드 시스템이 웜 본 팔레트를 명시적으로 지정한 경우에만 적용됩니다.

## 템플릿 선택

- **Pentagram Curatorial** (기본값, 번들 포함): 넉넉한 여백, 슬라이드당 단일 히어로 이미지, 절제된 타이포그래피, 푸터 브랜드 마크. Pentagram 케이스 스터디 슬라이드 작업의 형태를 따릅니다.
- **Wolff Olins Brand-Strategy** (대안): 브랜드 이미지와 함께 전략 다이어그램이 배치되는 구성으로, 브랜드 표면 아래에 전략적 내용을 담아야 하는 덱에 적합합니다.
- **Bibliothèque Editorial** (대안): 타이포그래피 밀도가 높고 매거진 그리드 레이아웃을 사용하며, 편집적인 느낌의 브랜드 덱(출판, 콘텐츠, 미디어 분야)에 적합합니다.

## 번들 덱을 시작점으로 활용하기

포함된 참조 덱은 실제 사례 브랜드에 대한 7-시스템 감사를 시연하는 완성된 16슬라이드 브랜드 적용 엔터프라이즈 피치 자료입니다. agent는 이 덱을 복사하여 슬롯 문법을 유지하면서 콘텐츠를 교체해야 합니다 — 브랜드 마크가 있는 표지, 브랜드 고정 아젠다, 팔레트 시그니처 why-now 슬라이드, 브랜드 그리드가 적용된 본문 슬라이드, 브랜드 데이터 시각화 슬라이드, 브랜드 사진 클로징, 브랜드에 맞는 연락처 카드가 포함된 크레딧, Brand-Decision Map 부록. 슬롯 문법은 고정이며, 브랜드 입력값만 교체됩니다.

## 권장 N-슬라이드 구조

일반적인 16슬라이드 브랜드 적용 엔터프라이즈 피치 (번들 참조 덱):

| # | 페이지 | 목적 |
|---:|---|---|
| 1 | 표지 — 브랜드 워드마크 + 브랜드 이미지 1장 | 무대 위의 브랜드 |
| 2 | 브랜드 고정 아젠다 — 타입 시스템 표시 | 1차 타이포그래피 검증 |
| 3 | Why-now 진술 — 단일 브랜드 컬러 블록 | 팔레트 시그니처 |
| 4–13 | 본문 슬라이드 — 기존 콘텐츠에 브랜드 그리드 적용 | 10개 슬라이드에 걸친 시스템적 적용 |
| 14 | 데이터 슬라이드 — 브랜드로부터 상속된 차트 팔레트 | 브랜드 데이터 시각화 검증 |
| 15 | 클로징 — 브랜드 사진 처리 방식 적용 | 사진 시그니처 |
| 16 | 크레딧 — 타입 조합 고정, 브랜드 연락처 카드 | 브랜드에 맞는 마무리 |
| 부록 | Brand-Decision Map — 슬라이드별 7-시스템 감사 | 시스템적 적용의 증거 |
